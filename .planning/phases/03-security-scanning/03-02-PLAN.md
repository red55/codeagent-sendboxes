Append a new `trivy-scan` job to `.github/workflows/pr.yml` (after the existing `build` job). The job structure:

```yaml
  trivy-scan:
    name: Security Scan (${{ matrix.variant }})
    needs: build
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    strategy:
      fail-fast: false
      matrix:
        variant: [oc-go, qc-go, oc-ansible, qc-ansible]
    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Run Trivy vulnerability scan
        uses: aquasecurity/trivy-action@v0.35.0
        with:
          image-ref: ghcr.io/red55/sandbox-${{ matrix.variant }}:pr-${{ github.event.number }}
          format: sarif
          output: trivy-vuln-results.sarif
          severity: CRITICAL,HIGH
          exit-code: "1"
          ignore-unfixed: true
          vuln-type: os,library
          config: .github/lint/trivy.yaml

      - name: Run Trivy secret scan
        uses: aquasecurity/trivy-action@v0.35.0
        with:
          image-ref: ghcr.io/red55/sandbox-${{ matrix.variant }}:pr-${{ github.event.number }}
          format: sarif
          output: trivy-secret-results.sarif
          scanners: secret
          config: .github/lint/trivy-secret.yaml
          ignorefile: .github/lint/trivy-ignore

      - name: Run Trivy misconfiguration scan
        uses: aquasecurity/trivy-action@v0.35.0
        with:
          scan-type: config
          scan-references: |
            base/Dockerfile
            base/opencode.Dockerfile
            base/qwencode.Dockerfile
            golang/Dockerfile
            ansible/Dockerfile
          format: sarif
          output: trivy-config-results.sarif
          config: .github/lint/trivy.yaml

      - name: Upload vulnerability SARIF to Security tab
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: trivy-vuln-results.sarif
          category: trivy-vuln-${{ matrix.variant }}

      - name: Upload secret scan SARIF to Security tab
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: trivy-secret-results.sarif
          category: trivy-secret-${{ matrix.variant }}

      - name: Upload config scan SARIF to Security tab
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: trivy-config-results.sarif
          category: trivy-config-${{ matrix.variant }}

      - name: Post Trivy findings as PR comment
        uses: reviewdog/action-trivy@v0
        if: always()
        with:
          github_token: ${{ secrets.github_token }}
          reporter: github-pr-review
          level: warning
          path: trivy-vuln-results.sarif
          filter_mode: nofilter
```

Key implementation details:
- Per D-01: PR builds scan locally-built image using `image-ref` with PR tag
- Per D-02: PR comments posted via reviewdog/action-trivy with `level: warning` (never blocks PR)
- Per D-06: Three separate Trivy steps — vulnerability scan, secret scan, config scan
- Per D-10: SARIF upload to Security tab for each scan type (vuln, secret, config)
- Per D-11: Secret SARIF uploaded with `if: always()` (not `if: failure()`) — Trivy v0.35+ handles empty SARIF gracefully
- Matrix matches build job: `[oc-go, qc-go, oc-ansible, qc-ansible]`
- `needs: build` ensures Trivy runs after images are built
- `exit-code: "1"` for vulnerability scan (CRITICAL/HIGH blocks), no exit-code for secret/config (warnings only)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI Runner → ghcr.io | Pulling images for Trivy scan requires registry access |
| CI Runner → Trivy DB | Downloads vulnerability DB from GitHub API (rate-limited, untrusted source before verification) |
| CI Runner → GitHub Security tab | SARIF upload requires security-events:write permission |
| PR event → PR comment | reviewdog posts comments as PR author (potential for comment spam) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Information Disclosure | Trivy scan results in SARIF | mitigate | SARIF uploaded to GitHub Security tab (repo-scoped, not public). Secret findings suppressed for known paths. |
| T-03-06 | Tampering | Workflow files | mitigate | Source-controlled, reviewed via PR. Matrix strategy prevents single-point failure. |
| T-03-07 | Elevation of Privilege | GITHUB_TOKEN in PR workflow | mitigate | pr.yml has `security-events: write` only (no `packages: write`). Cannot push images from PR. |
| T-03-08 | Spoofing | Trivy DB source | accept | Trivy uses official GitHub releases; DB integrity verified by aquasecurity. Rate limits managed by cron (plan 03). |
</threat_model>
