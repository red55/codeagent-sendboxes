Create `.github/lint/trivy.yaml` with the following content:

```yaml
# Trivy centralized configuration
# Referenced by: pr.yml, release.yml, trivy-cron.yml
#
# Per D-03: CRITICAL + HIGH block the build, MEDIUM is reported only
# Per D-04: Secret and misconfiguration findings are warnings only
# Per D-05: Suppress Context7 token in base/.opencode/opencode.json by file path
# Per D-06: Image scan (vuln+secret) + Dockerfile config scan
# Per REQ-SEC-001/002/003: All three scan types configured

# Severity levels that block the build (exit-code: 1)
severity:
  - CRITICAL
  - HIGH

# Vulnerability scanning settings
vulnerability:
  # Scan OS packages and language libraries
  vuln-type:
    - os
    - library
  # Ignore unfixed vulnerabilities (reduces noise from Debian/Ubuntu base images)
  ignore-unfixed: true

# Secret scanning settings
secret:
  # Enable secret detection in image layers
  config: .github/lint/trivy-secret.yaml

# Configuration / misconfiguration scanning
config:
  # Check Dockerfiles for security misconfigurations
  # Detects: NOPASSWD sudo, missing HEALTHCHECK, HTTP endpoints, etc.
  # Per D-04: These are warnings only, never block
  formats:
    - json

# Ignorefile for known false positives
# Per D-05: Suppress Context7 Bearer token in base/.opencode/opencode.json
# Trivy supports ignorefiles at: ~/.trivyignore or via --ignorefile flag
# We create a project-level ignorefile and reference it in workflows
ignorefile: .github/lint/trivy-ignore

# Timeout for scans (prevents hanging on large images)
timeout: 10m
```

Also create `.github/lint/trivy-ignore` (the ignorefile referenced above):

```
# Trivy ignore file — known false positives
# Per D-05: Suppress Context7 Bearer token in opencode.json
# Format: <rule-id> <path> <comment>
# For secrets, we suppress by file path pattern
# trivy-secret.yaml config handles the actual path-based suppression
```

And create `.github/lint/trivy-secret.yaml` (secret scanner config):

```yaml
# Trivy secret scanner configuration
# Per D-05: Suppress Context7 token in base/.opencode/opencode.json by file path
# Reference: https://aquasecurity.github.io/trivy/latest/docs/scanner/secret/

rules:
  # Default rules are enabled automatically (API keys, tokens, passwords)

# Custom ignore rules — suppress specific paths from secret scanning
# Per D-05: base/.opencode/opencode.json contains Context7 Bearer token (known issue)
# Using path-based suppression is more targeted than pattern matching
ignored_paths:
  - "base/.opencode/opencode.json"
```

Key design decisions implemented:
- D-03: `severity: [CRITICAL, HIGH]` — only these block the build
- D-04: Secret and config scanning configs are separate from severity (warnings only)
- D-05: `ignored_paths` in `trivy-secret.yaml` suppresses `base/.opencode/opencode.json`
- D-06: Separate `vulnerability`, `secret`, and `config` sections for each scan type
- D-12: Trivy action version pinned in workflow files (not in config)
- REQ-SEC-001: `vuln-type: [os, library]` and `ignore-unfixed: true`
- REQ-SEC-002: Secret scanning enabled with path-based suppression
- REQ-SEC-003: Config scanning section for Dockerfile misconfiguration checks
