Modify `.github/workflows/pr.yml` to add fork PR-gated image push:

1. **Update permissions block** — Add `packages: write` to the top-level permissions:
```yaml
permissions:
  contents: read
  packages: write
  security-events: write
```

2. **Add GHCR login step** (gated on fork PR) after the build step in the build job:
```yaml
      - name: Login to GHCR (fork PRs only)
        uses: docker/login-action@v4
        if: github.event.pull_request.head.repo.fork == true
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
```

3. **Add image push step** (gated on fork PR) after login:
```yaml
      - name: Push PR image to GHCR (fork PRs only)
        uses: docker/build-push-action@v7
        if: github.event.pull_request.head.repo.fork == true
        with:
          context: .
          push: true
          tags: ghcr.io/red55/sandbox-${{ matrix.variant }}:pr-${{ github.event.number }},ghcr.io/red55/sandbox-${{ matrix.variant }}:${{ github.sha }}
          cache-from: type=gha
```

The build job structure becomes:
```yaml
  build:
    name: Build (${{ matrix.variant }})
    needs: lint
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        variant: [oc-go, qc-go, oc-ansible, qc-ansible]
    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v4

      - name: Set up QEMU
        uses: docker/setup-qemu-action@v4

      - name: Build image
        run: |
          make pull
          make all
        env:
          GHCR_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          VER: ${{ github.sha }}

      - name: Login to GHCR (fork PRs only)
        uses: docker/login-action@v4
        if: github.event.pull_request.head.repo.fork == true
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Push PR image to GHCR (fork PRs only)
        uses: docker/build-push-action@v7
        if: github.event.pull_request.head.repo.fork == true
        with:
          context: .
          push: true
          tags: ghcr.io/red55/sandbox-${{ matrix.variant }}:pr-${{ github.event.number }},ghcr.io/red55/sandbox-${{ matrix.variant }}:${{ github.sha }}
          cache-from: type=gha
```

- Push tags include both `pr-<number>` (for Trivy scanning) and `sha` (for traceability)
- `docker/build-push-action@v7` loads the locally-built image and pushes it (no rebuild)
- Lint and Trivy scan jobs continue to run for all PRs

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fork PR → GHCR push | Untrusted fork code could push malicious images — gated by fork check |
| CI Runner → GHCR (login) | GITHUB_TOKEN used for authentication, scoped to repo |
| PR event → Image tags | Tags derived from github.event.number (PR number) — must not be spoofed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Spoofing | Fork PR check bypass | mitigate | Fork check uses `github.event.pull_request.head.repo.fork` — this is set by GitHub Actions runner, not user input. Cannot be spoofed. |
| T-04-06 | Elevation of Privilege | GHCR push from untrusted code | mitigate | Push steps gated on `if: github.event.pull_request.head.repo.fork == true`. Non-fork PRs cannot push. |
| T-04-07 | Information Disclosure | PR image tags in registry | accept | PR tags (pr-<number>) are visible in registry. Images contain build artifacts but no secrets (lint/build jobs don't inject secrets). |
| T-04-08 | Tampering | Build job for non-fork PRs | mitigate | Non-fork PRs still build and scan images locally — no push, but Trivy scanning still runs. |
</threat_model>
