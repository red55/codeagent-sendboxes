# Milestone v0.35.0: Sandbox CI/CD Pipeline

**Status:** ✅ SHIPPED 2026-04-21
**Phases:** 1-4
**Total Plans:** 11

## Overview

Built a complete CI/CD pipeline for AI agent sandbox Docker images using GitHub Actions. The pipeline lints Dockerfiles, builds 4 image variants (opencode/qwencode × golang/ansible), runs Trivy security scans, pushes to GHCR with SBOM attestations and build provenance, and supports fork PR builds with gated registry access.

## Milestone Summary

**Key Decisions:**
- Use GitHub Actions with native ghcr.io integration
- 4-phase rollout for clarity (foundation → reliability → security → publishing)
- GHA cache for builds (simpler than registry cache)
- Trivy for security scanning (SARIF output integrates with GitHub Security tab)
- `make all` in 4 matrix jobs (preserve Makefile, gain CI parallelism)
- Separate pr.yml + release.yml (cleaner separation of concerns)
- Strict hadolint + .hadolint.yaml (fail on warnings, suppress false positives)
- CycloneDX SBOM format, mode=max provenance
- Fork PR gating for registry push (pr-<number> tags)

**Issues Resolved:**
- Build reproducibility (SOURCE_DATE_EPOCH, .dockerignore, version pinning)
- Security scanning integration (Trivy vulnerability, secret, misconfiguration)
- SBOM and provenance attestation for published images
- Fork PR build isolation from registry

**Technical Debt:**
- Multi-platform (arm64) support deferred
- cosign image signing deferred
- Environment protection rules deferred
- healthcheck.sh not integrated into `make check` (requires pre-built images)
